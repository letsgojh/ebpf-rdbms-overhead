# 의사결정 로그

프로젝트 진행 중 내린 굵은 의사결정을 기록한다. 코드나 git 커밋만 봐서는 "왜 이렇게 했는지"가
안 드러나는 것들(Umbra go/no-go, 벤치마크 변경 이력 등) 위주로 남긴다. 최신 항목을 위에 추가한다.

## 2026-07-23 — Umbra 심볼 접근성 미확정에 따른 범위 조정

**배경.** Umbra 바이너리에 uprobe/USDT로 붙일 수 있는 심볼이 안정적으로 노출돼 있는지 아직
확인되지 않았다.

**결정.**
- Group A-1/A-2(격리 floor 측정, 앰비언트 부하 민감도)는 `harness_floor.c` 기반 syscall 레벨
  측정이라 Umbra 자체 심볼에 의존하지 않으므로 **그대로 진행**한다.
- Umbra 자체 함수에 uprobe/USDT를 붙여야 하는 Group(D: 유저스페이스 vs 커널스페이스, E: 쿼리
  플랜 민감도 등)은 접근성 확정 전까지 **커널 레벨(syscall 진입/종료)로 근사**한다
  (`docs/01_experiment_design.md` "구현 가능한 probe 위치" 표 참고).

**영향.** Umbra의 D/E 결과는 다른 시스템과 비교할 때 "커널 레벨 근사치"라는 전제를 달고
해석해야 한다. 심볼 접근성이 확정되면 재측정 여부를 다시 검토한다.
